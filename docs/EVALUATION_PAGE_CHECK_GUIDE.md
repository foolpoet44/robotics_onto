# 평가 전용 도메인 페이지 직접 확인 가이드

이 문서는 `/evaluation` 페이지 변경사항을 로컬에서 직접 확인하는 절차를 정리합니다.
평가 전용 페이지는 도메인 영역만 분리해 보여주는 것이 목적이므로, 아래 항목을 중심으로 검증합니다.

## 1. 개발 서버로 화면 확인

```bash
npm install
npm run dev
```

브라우저에서 다음 주소로 접속합니다.

```text
http://localhost:3000/evaluation
```

확인할 항목은 다음과 같습니다.

- 상단 제목이 `도메인 분류 평가 페이지`로 표시되는지 확인합니다.
- 평가 대상 도메인 카드가 6개 표시되는지 확인합니다.
- 각 카드가 도메인명, 영문명, 설명, 스킬 수와 함께 중요도 평가(1~5점) 기능을 제공하는지 확인합니다.
- 평가자명을 입력한 뒤 도메인별 중요도 점수를 선택하고 `평가 저장`을 누르면 평균 중요도와 최근 평가 목록이 갱신되는지 확인합니다.
- 스킬 상세 목록, 조직 정보, 개인 역량 정보, 육성 트랙, 검수 큐가 평가 카드 안에 표시되지 않는지 확인합니다.
- 하단의 `평가 목적 적합성 검증`과 `보완사항` 섹션은 화면에서 제거되었으며, 해당 내용은 `docs/PROJECT_IMPROVEMENTS.md`로 이관되었는지 확인합니다.

## 2. 프로덕션 빌드로 라우트 생성 확인

```bash
npm run build
```

빌드 출력의 `Route (app)` 목록에 다음 라우트가 포함되어야 합니다.

```text
/evaluation
```

이 라우트가 표시되면 Next.js 정적 빌드 대상에 평가 페이지가 포함된 것입니다.

## 3. 타입 검사 확인

```bash
npm run type-check
```

타입 오류 없이 종료되면 평가 페이지에서 사용하는 도메인 데이터와 서버 데이터 로딩 코드가 타입 수준에서 유효한 것입니다.

## 4. 내비게이션 진입 확인

개발 서버 실행 후 브라우저에서 아무 페이지에 접속한 뒤 상단 내비게이션의 `평가` 메뉴를 클릭합니다.
클릭 후 주소가 `/evaluation`으로 이동하고 평가 전용 페이지가 표시되는지 확인합니다.

## 5. 평가 목적 적합성 체크리스트

| 검증 항목 | 기대 결과 |
| --- | --- |
| 도메인만 표시 | 6개 도메인 평가 카드만 표시 |
| 부가 정보 제외 | 스킬 상세, 조직, 역량, 육성, 검수 데이터 미표시 |
| 평가 안내 | 평가 전용 목적과 도메인별 중요도 평가 안내 표시 |
| 중요도 평가 기능 | 평가자명 입력 후 도메인별 1~5점 평가 저장 및 평균/최근 평가 표시 |

적합성 점검 결과와 후속 개선 항목은 `docs/PROJECT_IMPROVEMENTS.md`에서 관리합니다.
