# 후보자 개인별 프로파일 구축안 (업계 벤치마킹 기반)

육성 트랙(`/development-tracks`)에 후보자 개인별 프로파일 기능을 추가하기
위한 설계안입니다. 업계 탑 탤런트/스킬 플랫폼의 프로파일 구성요소를
벤치마킹하고, 본 프로젝트의 온톨로지·4대 도메인·평가 인프라에 맞게
재구성했습니다.

## 1. 벤치마킹 — 업계 탑 솔루션의 프로파일 구성

| 솔루션 | 프로파일 핵심 기능 | 본 프로젝트 적용 포인트 |
| --- | --- | --- |
| **Workday** (Skills Cloud + Talent Profile) | 스킬+숙련도 통합 프로파일, 자격/인증, 경력 이력, 성장 계획, 매니저 뷰 | 온톨로지 148 스킬 기준의 보유/목표 숙련도 프로파일, 레벨 인증(AX Starter~Expert) 표시 |
| **SAP SuccessFactors** (Talent Card + CDP) | 한 장짜리 탤런트 카드 요약, 개발 목표(CDP), 9-box, 배지 | 프로파일 상단 "탤런트 카드"(단계·도메인·준비도 요약), 단계별 개발 목표 |
| **Degreed** | 스킬 등급의 다중 출처(자기/동료/상사/추론), 스킬 인증, 학습 이력 통합 | 스킬 진단에 **검증 출처**(자기신고/멘토/부트캠프/평가 워크벤치) 도입 — 근거 있는 숙련도 |
| **Eightfold AI** | 보유 스킬 기반 인접 스킬·커리어 패스 추천, 역할 적합도 % | 온톨로지 관계망(`prerequisite`/`related_skills`)으로 **다음 학습 스킬 추천** — AI 없이 그래프 휴리스틱으로 동일 효과 |
| **Gloat / Fuel50** (Talent Marketplace) | 프로젝트/기그 매칭, 성장 경로 시각화, 멘토 매칭 | `1人1案` 현장 임팩트 과제를 프로젝트 카드로, 멘토를 평가자 명부와 연계 |

**공통 분모(업계 표준 프로파일의 5요소)**: ① 탤런트 카드 요약 ② 검증
출처가 붙은 스킬 프로파일 ③ 갭 분석·준비도 ④ 성장 여정 타임라인
⑤ 다음 행동 추천. 본 구축안은 이 5요소를 그대로 화면 골격으로 삼습니다.

## 2. 프로파일 화면 설계 (5블록)

경로: `/development-tracks/[trackId]/candidates/[candidateId]` (기존 상세
페이지를 프로파일로 승격).

```
┌ A. 탤런트 카드 ─────────────────────────────────────┐
│ 후보자ID · 전문영역(4대 도메인) · 현재 단계 · 목표(Lv3)  │
│ 종합 준비도 게이지(%) · 멘토 · 코호트                  │
├ B. 스킬 프로파일 ──────────────────────────────────┤
│ 중간분류별 그룹핑된 스킬 목록: 현재/목표 레벨 바,        │
│ 검증 출처 배지(자기신고·멘토·부트캠프·워크벤치),         │
│ 갭 스킬 강조                                         │
├ C. 역량·갭 분석 ───────────────────────────────────┤
│ 5축 직능 역량 레이더 + 도메인별 스킬 갭 요약             │
├ D. 성장 여정 타임라인 ─────────────────────────────┤
│ 단계 이력(승인자·일자) + 레벨 인증 + 1人1案 과제 카드    │
├ E. 다음 행동 추천 ─────────────────────────────────┤
│ 갭 상위 스킬 + 선수관계 미충족 스킬 → 추천 학습 순서     │
└──────────────────────────────────────────────────┘
```

### 준비도(readiness) 산식 — 탤런트 카드의 핵심 지표

```
준비도 = 0.4 × 스킬 충족률(현재레벨 ≥ 목표레벨인 스킬 비율)
       + 0.3 × 단계 진행률(완료 단계 수 / 전체 단계 수)
       + 0.3 × 역량 평균(5축 평균 / 5)
```

가중치는 육성위원회가 조정할 수 있도록 트랙 JSON에 둡니다. Eightfold의
"역할 적합도 %"에 해당하며, 산식이 투명하다는 점이 차별점입니다.

### 다음 스킬 추천 로직 (Eightfold 인접 스킬의 온톨로지 구현)

1. 갭 스킬(현재 < 목표) 중 `prerequisite` 관계의 선행 스킬이 이미 충족된
   것을 우선 추천.
2. 보유 스킬의 `related_skills`(co_required/specialization) 중 아직
   프로파일에 없는 스킬을 "확장 후보"로 제시.
3. 후보자의 4대 도메인과 다른 도메인의 `cross_domain` 스킬 1~2개를
   "크로스 도메인 추천"으로 제시(허브인 Data Intelligence 우선 — 레벨
   선수 체계와 일치).

## 3. 데이터 모델 확장

기존 `sample-candidates.json` 스키마를 하위 호환으로 확장합니다
(기존 필드 무변경, 신규 필드 optional).

```jsonc
{
  "candidate_id": "CAND-001",
  // ...기존 필드 유지...
  "profile": {
    "career_goal": "AMR 함대 운영 Tech Leader",
    "mentor_evaluator_id": "EVAL-101",        // 평가자 명부 연계
    "certifications": [
      { "level_id": "physical-ai-lv1", "certified_at": "2026-02-10" }
    ],
    "learning_history": [
      { "title": "통합 부트캠프", "type": "bootcamp", "completed_at": "2026-03-28" }
    ]
  },
  "skill_assessments": [
    {
      "skill_id": "RSF-AMR-001",
      "current_level": 3,
      "target_level": 3,
      "source": "bootcamp",                    // self|mentor|bootcamp|workbench
      "assessed_at": "2026-03-28"
    }
  ]
}
```

- **검증 출처 자동 연계**: 스킬 평가 워크벤치(`skill_evaluation_labels`)에
  후보자 본인이 평가자로 로그인해 남긴 기록, 그리고 향후 개인 역량
  진단(`employee-competency-assessments`)을 프로파일에 `workbench` 출처로
  표시할 수 있습니다(읽기 연계, 데이터 복제 없음).
- 갱신은 기존 관행대로 스크립트 경유:
  `update:development-candidate`에 `--skill`, `--skill-level`, `--skill-source`,
  `--mentor`, `--certify` 옵션을 추가하고 저장 전 무결성 검증(스킬 실존,
  레벨 범위, 평가자 실존)을 수행합니다.

## 4. 개인정보 경계 (기존 운영 원칙 승계)

`docs/DEVELOPMENT_TRACK_OPERATIONS.md`의 원칙을 그대로 따릅니다.

- MVP는 **익명 ID(CAND-XXX)** 유지. 실명·사번은 정적 JSON에 저장하지
  않으며, 실제 인사정보 연결은 운영 전환 체크리스트(권한·보존기간·감사
  로그·HRIS 연동) 확정 이후에만 진행합니다.
- 접근 제어(운영 전환 시): 평가자 세션 인프라 재사용 —
  본인(조회) / 멘토·육성위원회(갱신) 역할 구분. 프로파일 페이지는 현재
  MVP에서는 조회 전용이고 모든 변경은 스크립트+검증 경유입니다.

## 5. 구현 로드맵

| 단계 | 내용 | 산출물 |
| --- | --- | --- |
| P1 데이터 | 스키마 확장(profile·source 필드) + 샘플 데이터 보강 + `validate:development` 검증 규칙 추가 + 갱신 스크립트 옵션 확장 | 데이터·스크립트·테스트 |
| P2 프로파일 화면 | 탤런트 카드·스킬 프로파일(중간분류 그룹)·역량 레이더·여정 타임라인·1人1案 카드 | 후보자 상세 페이지 개편 |
| P3 지표·추천 | 준비도 산식 구현(가중치는 트랙 JSON) + 온톨로지 그래프 기반 다음 스킬 추천 | 준비도 게이지·추천 블록 |
| P4 연계·권한 | 워크벤치/역량 진단 기록 읽기 연계, 평가자 세션 기반 접근 구분 | 출처 배지 자동화 |

P1~P3는 현 저장소 안에서 완결되며 외부 인프라가 필요 없습니다. P4의
실명 연결만 운영 전환 체크리스트와 함께 진행합니다.
