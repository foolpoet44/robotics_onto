# 전문가 검토 회의록 — Agentic AI · Data Intelligence · Physical AI 분류체계

> 성격: AI Factory 육성체계(스킬 온톨로지·커리큘럼)에 대한 **전문가 검토 회의**의 정리
> 기록물. 이 문서는 SSOT·온톨로지·발행물이 아니라 **의사결정 근거를 남기는 감사
> 기록**이다. 여기의 판정은 곧바로 데이터에 반영되지 않으며, 각 항목은 이후 정식
> 경로(생성기 수정→`npm run generate:data`, 오버라이드 장부, 커리큘럼 재발행,
> 평가 페이지 시그널)로 처리한다.
>
> - 정리 시점: 2026-07
> - 선행 회의: 고민석 팀장(Digital Twin) 인터뷰 → DT 7단계 워크플로우 반영(머지 완료)
> - 이번 회의 주제: Agentic AI · Data Intelligence · Physical AI 분류체계 검토

## 1. 과제의 성격 (전제 재확인)

- **타겟은 완성형 전문가가 아니라 선임급(주니어) 연구원**이다. 이들에게 전문가로
  성장할 수 있는 **비전 로드맵**을 제시하는 것이 목적이며, 육성 로드맵을 논의할
  **공통 언어(내부 terminology) 통일**이 1차 목표다.
- 1차 대상: **선임급 핵심 인재 약 30명**. 트랙 1개씩 선택 → 한 사이클 검증 →
  정합성 확인 후 기존 **R&D 칼리지** 형태로 전 연구원 대상 배포.
- 체계에는 **필요한 스킬·지식을 빠짐없이(MECE) 반영하되, 전부 교육과정화하지는
  않는다.** "필요 시 언제든 최단기간에 교육으로 공급할 수 있는 Ready 상태" 확보가
  목표. 필수학점식 운영은 하지 않고 선택 기반, 트랙 전환 시 리스킬링 교육 지원.
- 부수 아이디어: 강사·보조강사 역할의 **교육 에이전트화**(고민석 팀장 제안).
  전문가 반응은 "범용 LLM 수준은 가능, 우리만의 것을 넣는 건 어렵다"로 유보적.

## 2. Agentic AI — 영역별 → **업무 플로우형**으로 재편 (★핵심 결정)

현재 초안은 품질·물류·설비 등 **영역별 에이전트 나열**이다. 전문가 의견은 명확한
반대: **"품질 에이전트든 설비 에이전트든 만들 때 필요한 스킬은 거의 같다. DT처럼
개발 과정(플로우)으로 재편해야 한다."** 영역 구분은 심화(도메인 스페시픽) 단계에서
의미가 있고, 기초는 공통 플로우로 통일하는 것이 맞다.

### 2-1. 단일 에이전트 개발 6단계 플로우

| 단계              | 내용                                                                   |
| ----------------- | ---------------------------------------------------------------------- |
| ① 문제 정의       | 요구사항 정리 + 기존 업무 워크플로우 + 데이터 플로우 정의              |
| ② 파이프라인 설계 | 데이터 파이프라인 설계(에이전트화 대상 확정)                           |
| ③ 툴·데이터 확보  | 수집 체계 강화, 툴 확보/개발(모델 매칭·최적화·비즈니스 로직), LLM 선택 |
| ④ 구현·검증       | 구동, 속도·반응·정확도 검증                                            |
| ⑤ 성능 모니터링   | 성능 모니터링 및 문제 피드백 수집                                      |
| ⑥ 고도화          | 피드백 기반 성능 고도화                                                |

> "이 6단계를 한 바퀴 돌면 1년 차가 만들어진다"(전문가 표현). 기존 초안의
> "에이전트 설계 및 오케스트레이션"은 기초로 두기엔 난이도 스펙트럼이 너무 넓어
> (프롬프트 설계~멀티에이전트 오케스트레이션 논문 수준) 재배치 필요.

### 2-2. 멀티 에이전트 (별도 후속 단계)

단일 에이전트가 많아진 뒤의 상위 개념으로 분리한다.

1. 멀티 에이전트 아키텍처링(계층형/수평형, 정책·권한 설계)
2. 아키텍처 기반 멀티 에이전트 구현
3. 멀티 에이전트 성능 검증

### 2-3. 실무(기초) vs 심화

- **실무(기초)**: 에이전트를 구현할 줄 아는 공통 기술 역량.
- **심화**: 도메인 스페시픽 에이전트. "설비를 모르면 설비 에이전트를 못 만들고,
  품질 관리를 모르면 품질 에이전트를 못 만든다" — 적용 도메인 지식이 전제.

## 3. Data Intelligence — Agentic AI와 **통합** 제안 (구조 변경, 미확정)

전문가 핵심 주장: **분석·예측·인텔리전스는 에이전트의 '툴'이고, 데이터 패브릭·
지식은 에이전트를 구성하는 '데이터'다 → DI는 Agentic과 결합하는 것이 맞다.**

- DI 항목들(데이터 전처리·파이프라인·지식베이스 구축 등)은 **에이전트 육성의
  실무(기초) 단계로 매핑**되는 것이 자연스럽다.
- 센서 데이터 수집·통신은 이미지 처리와 성격이 겹치고, 예지 분석도 중복 소지.
  실무 조언자도 "DI는 과거 DX 교육(파이썬 등)과 중복"이라고 동조.
- 단, 발언자 스스로 **"원장님이 AI Factory를 3축으로 정리해 둔 것을 우리가 바꾸는
  것"**이라는 단서를 달았다 → **확정이 아닌 제안.** 4대(칼리지) 구조 자체의 변경은
  파급이 크므로(칼리지 매핑·오버라이드·발행물 전반) 방향성 확인 전 착수 보류.

## 4. Physical AI — 커리큘럼 레벨링 + 스킬 제외·통합 판정

### 4-1. 레벨 구조

| 레벨 | 내용                                                                                          |
| ---- | --------------------------------------------------------------------------------------------- |
| 초급 | 로봇·비전·AMR의 **하드웨어/구조/키네마틱스** + 각 장비 산출 데이터(축·속도·SLAM 등) 기초 이론 |
| 중급 | 산출 데이터의 처리·변환·전처리, 인식 모델, 경로 생성 등 **AI 모델**                           |
| 고급 | **실습·적용 사례 중심**(구조 암기가 아니라 적용)                                              |

### 4-2. RFM/VLA 취급 (중요)

- **RFM/VLA는 "고급보다도 위".** 구조를 가르치는 것은 무리이며, **최신 사례·동향
  수준으로만** 다룬다(휴머노이드 등 사례 소개 → 기술영업/도메인 전문가로서 대화
  가능한 수준). 내부 구조 해설은 학습 효용이 낮다는 판정.

### 4-3. 개별 스킬 판정 (커리큘럼 관점)

| 스킬/영역                                    | 판정                                                                |
| -------------------------------------------- | ------------------------------------------------------------------- |
| 좌표계 설정·보정, 로봇 프로그래밍, 로봇 제어 | 유지하되 **앞단(기초) 수준만**                                      |
| 로봇 티칭 방법론                             | **유지**(현장 티칭 실무)                                            |
| 로봇 안전                                    | **유지**(교육 필수)                                                 |
| 협력업체 기술 지원                           | **제외**(사전 스캐폴딩, 불필요)                                     |
| 사이클타임 달성                              | 현장 적응 영역(교육 우선순위 낮음)                                  |
| PLC·설비 제어 연동                           | **제외 의견**(로봇 커리큘럼과 무관)                                 |
| 협동로봇 (분류 전체)                         | **분류를 통으로 빼고** 로봇 기초 소개에 흡수(산업용·협동 함께 언급) |
| 힘/토크 모니터링 등                          | **너무 세분화 — 묶어라**(재정리 후 재평가)                          |
| SLAM                                         | 자율운영의 위치인식 기술 → GPS 등과 함께 **그룹핑 필요**            |

> ⚠️ 위 "제외/통합"은 **커리큘럼(교육 콘텐츠) 관점의 취사선택**이지 온톨로지 스킬의
> 물리적 삭제가 아니다. `skill_id` 불변 원칙에 따라 온톨로지 레코드는 유지하고,
> 발행물(커리큘럼)에서 빼거나 표시 클러스터로 묶는 방식으로 반영한다.

## 5. 육성 구조 — T자형 성장 + 트랙 이원화 (합의)

- **성장 방향 논쟁**: (a) 3개 도메인을 모두 아는 AX 스페셜리스트(팔란티어 FD 비유)
  vs (b) 한 트랙을 완성한 뒤 수평 확장. → **(b)로 수렴**: "하나의 트랙에 뿌리를
  내려(혼자 프로젝트를 수행할 수 있는 수준까지) → 이후 수평 확장." 이론적 넓히기만
  먼저 하면 실무 진입까지 시간이 과다하다는 실무 조언자 지적 반영.
- **트랙 이원화 합의**: **기술 트랙(테크니션)**과 **비즈니스 리더 트랙(PM·프리세일즈·
  컨설팅)**을 분리. 팔란티어 FD는 프리세일즈이고 상세 뎁스는 백단 전문가가 백업한다는
  논리. "우리가 양성하려는 사람이 앞단(FD)인지 백단(전문가)인지 명확해야 스킬을
  줄 수 있다"는 지적에 운영 주관자 전적 동의.
- 운영 방식: 선임급 핵심 인재는 **트랙 1개 선택**(나중에 전환 가능, 전환 시
  리스킬링 교육 제공). 필수/선택 학점식 학사 운영은 하지 않는다.

## 6. 후속 액션 (회의에서 확정)

1. **Agentic 분류를 6단계 플로우 + 멀티에이전트 3단계로 재정리** → 전문가 재검토.
2. **로봇(PA) 영역 재정리 → 박석우 책임에게 평가 페이지(1~5점 + 삭제 태그)로 평가
   요청.** 10분 내 커버 가능한 분량으로 축소.
3. **DI–Agentic 통합안 조정**(방향성 확인 후).
4. 오프닝 특강(최신 트렌드 중심) 후보: 서울대 **유영재** 교수(김대환 위원 추천),
   서울대 **윤병동** 교수(AI 네이티브 팩토리 — 에이전트 관점).
